3. (Tuỳ chọn) CNEE & NOTIFY PARTY — chỉ cho lô KHÔNG phải CN

Lô CN (factory=CN) luôn tự động điền Consignee/Notify Party theo đúng cửa hàng — không cần đụng vào phần này. Với các nước/factory khác (POP, SBGEAR, QIFENG, JION...), gõ vào 2 ô dưới đây 1 lần trước khi bấm chạy — tool sẽ tự điền thẳng vào file kết quả, khỏi phải sửa tay lại trong Excel mỗi lần. Để trống thì file sẽ có ghi chú nhắc điền tay.

Cần chọn: PDF (thư mục hoặc zip) + file DIM + Master Data.

Cách dùng

1) Chọn thư mục chứa toàn bộ PDF Packing List của 1 lô hàng (hoặc zip lại rồi upload file zip).
2) Chọn file DIM/khối lượng và file Master Data (SKU ↔ HS Code) tương ứng lô đó.
3) Nếu lô KHÔNG phải CN, có thể điền CNEE/Notify Party 1 lần ở mục 3 (tuỳ chọn) — lô CN thì bỏ qua, tự động rồi.
4) Bấm "Chạy OCR + Tách nhóm" — log xử lý hiện trực tiếp bên dưới.
5) Xong sẽ có 1 nút tải về 1 file .zip gồm: PL_Total.xlsx (2 sheet Packing List + Match_Status), cùng thư mục PL_SPLIT_OUTPUT/ (tách theo Factory / CN Port / CN Store, kèm PL_SPLIT_CONTROL.csv và báo cáo đối soát).
Shipping Mark trong Packing List tự lấy theo tên file PDF (bỏ đuôi .pdf). PO No. / Invoice No. / 中国标签名称 vẫn để trống — điền tay sau.
Logic OCR/tách nhóm giữ nguyên 100% so với bản Jupyter notebook gốc đã test — trang này chỉ là lớp giao diện chạy cùng logic đó ngay trong trình duyệt.
